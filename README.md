# Binance + Bybit Futures — мониторинг импульса цены (v3)

Бот мониторит **весь рынок** фьючерсов Binance и Bybit (USDT-пары, без привязки к топ-N)
и шлёт сигнал в Telegram, когда любая пара даёт импульс ≥30% за скользящее окно 24 часа —
в любую сторону. После первого сигнала продолжает сигналить каждые +10% (40%, 50%, 60%...).

## Правило двух бирж (без дублей)

- Если токен торгуется **на Binance** — мониторится только через Binance.
- Если токен есть **только на Bybit** (на Binance не листится) — мониторится через Bybit.
- Совпадающие тикеры никогда не дублируются между биржами.
- Каждый алерт содержит явную пометку биржи-источника и прямую ссылку на нужную биржу:
  `binance.com/en/futures/{symbol}` или `bybit.com/trade/usdt/{symbol}`.

## Архитектура

| Модуль | Назначение |
|---|---|
| `config.py` | Все настраиваемые параметры (обеих бирж) |
| `storage.py` | SQLite: подписчики + состояние алертов (переживает перезапуск) |
| `fetcher.py` | Список торгуемых пар Binance (весь рынок, фильтр по мин. объёму) |
| `bybit_fetcher.py` | То же самое для Bybit |
| `collector.py` | WebSocket-стримы Binance `kline_1m` |
| `bybit_collector.py` | WebSocket-стримы Bybit `kline.1.{symbol}` (свой протокол подписки) |
| `analyzer.py` | Скользящее окно цен, детектор импульса с уровнями, поле `exchange` в сигнале |
| `notifier.py` | Отправка/редактирование сообщений в Telegram, динамическая ссылка под биржу |
| `commands.py` | `/start`, `/stop`, `/status` — подписка через long polling |
| `main.py` | Точка входа: делит символы между биржами, запускает оба коллектора параллельно |
| `daily_report.py` | Ежедневный отчёт топ-10 рост/топ-10 падение (20:00 Киев/Москва) |

## Ежедневный отчёт топ-10

Раз в день, в 20:00 по времени Киева/Москвы (17:00 UTC), бот собирает дневное изменение
цены (от 00:00 UTC до момента отчёта) по всем отслеживаемым парам обеих бирж и присылает
подписчикам топ-10 пар с наибольшим ростом и топ-10 с наибольшим падением — с прямыми
ссылками на каждую пару.

В отличие от индикатора импульса (который смотрит на скользящее окно 24 часа),
этот отчёт использует обычное дневное изменение цены — `(текущая - open дня) / open дня`,
взятое из дневной свечи (`klines interval=1d`), а не из скользящего 24h-тикера.

Настраивается через `.env`:
- `DAILY_REPORT_HOUR_UTC` / `DAILY_REPORT_MINUTE_UTC` — время отправки (UTC)
- `DAILY_REPORT_TOP_N` — сколько пар показывать в каждом топе (по умолчанию 10)

## Как работает детектор импульса (ключевая логика)

Окно **скользящее**, а не фиксированными свечами. На каждой минутной свече бот сравнивает
текущую цену с ценой 24 часа назад — это значение постоянно "сдвигается" вперёд.

Это значит: если рост растягивается на несколько получасовых интервалов (например, +15% в один
условный получасовой период, затем ещё +20% в следующий), бот всё равно поймает момент, когда
суммарное изменение за последние 24 часа пересечёт 30% — независимо от того, как давно
начался рост. Фиксированные свечи с обнулением счётчика каждые 24ч такое движение
пропустили бы полностью.

```
Способ А (отклонён) — обнуление каждые 24ч:
  |--- +15% ---|--- +20% ---|   -> ни один кусок не дошёл до 30%, бот молчит

Способ Б (используется) — скользящее окно:
  |---------- +35% за любые скользящие 24ч ----------|   -> сигнал в момент пересечения
```

## Уровни сигналов

- **30%** — первый сигнал (новое сообщение в Telegram)
- **40%, 50%, 60%...** — повторные сигналы (редактирование того же сообщения, не спам)
- Если импульс откатывается ниже 30% — состояние сбрасывается, монета может сигналить заново

## Защита от Telegram rate limit

Вместо отправки нового сообщения на каждый уровень, бот **редактирует** существующее сообщение
по той же монете (`editMessageText`). Это и решает проблему читаемости, и резко снижает число
API-вызовов при массовых движениях рынка. Дополнительно — очередь отправки с паузой между
сообщениями и автоматический retry при ошибке 429.

## Быстрый старт

```bash
pip install -r requirements.txt
cp .env.example .env
# Вставьте TELEGRAM_TOKEN в .env
python main.py
```

После запуска подпишитесь на бота командой `/start` в Telegram.

## Команды бота

| Команда | Действие |
|---|---|
| `/start` | Подписаться на алерты |
| `/stop` | Отписаться |
| `/status` | Текущие настройки и статус подписки |

## Настройки (.env)

| Параметр | По умолчанию | Описание |
|---|---|---|
| `IMPULSE_START_THRESHOLD` | `30.0` | Порог первого сигнала, % |
| `IMPULSE_STEP` | `10.0` | Шаг повторных сигналов, % |
| `WINDOW_MINUTES` | `1440` | Скользящее окно расчёта, минут (1440 = 24ч) |
| `MIN_DAILY_VOLUME_USDT` | `1500000` | Мин. дневной объём — отсекает мёртвые пары |
| `SYMBOLS_REFRESH_SEC` | `86400` | Частота пересборки списка пар и перезапуска WS (сек, 86400 = 24ч) |

## Важные технические детали

- **WebSocket-соединения**: Binance ограничивает ~200 потоков на соединение. При мониторинге
  всего рынка (обычно 250-350 активных USDT-пар) бот автоматически открывает несколько
  параллельных соединений по 150 пар каждое.
- **Персистентность**: состояние активных импульсов и список подписчиков хранятся в SQLite
  (`bot_state.db`). При перезапуске бот восстанавливает состояние и не теряет историю — повторного
  спама после деплоя не будет.
- **Ежедневная пересборка списка пар**: раз в `SYMBOLS_REFRESH_SEC` (по умолчанию 24ч) бот
  заново запрашивает список торгуемых пар на обеих биржах и **реально перезапускает**
  WebSocket-подписки с обновлённым списком — а не просто обновляет список в памяти. Пары,
  набравшие объём выше фильтра, начинают мониториться; пары, упавшие ниже фильтра или
  делистнутые, отключаются. Накопленная 24-часовая история цен (`analyzer.py`) при этом
  НЕ сбрасывается — пересобираются только сетевые подключения, не данные анализа.

## Деплой на сервер (systemd)

```bash
sudo nano /etc/systemd/system/volbot.service
```

```ini
[Unit]
Description=Binance Impulse Bot
After=network.target

[Service]
WorkingDirectory=/path/to/binance-vol-bot
ExecStart=/usr/bin/python3 main.py
Restart=always
RestartSec=10

[Install]
WantedBy=multi-user.target
```

```bash
sudo systemctl enable volbot
sudo systemctl start volbot
sudo systemctl status volbot
```
